# Design Docs

This folder contains durable design notes for the Codex-ready hello-world
template.

Start here:

- `core-beliefs.md`: project beliefs that should remain stable across changes.
- `../references/codex-workflow.md`: official Codex workflow sources and how
  they map to this repository.
- `../exec-plans/README.md`: when and how to write task execution plans.

Design docs should explain decisions that affect future work. Keep transient
task notes in `docs/exec-plans/active/` while work is ongoing, then move useful
summaries to `docs/exec-plans/completed/`.
