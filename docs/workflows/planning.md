# Planning Workflow

Use planning for broad, ambiguous, risky, or architecture-sensitive work.

1. Clarify the goal and done condition.
2. Inspect relevant files, docs, tests, configs, schemas, and existing usage.
3. Identify constraints, risks, and validation commands.
4. Break work into small implementation steps.
5. Keep the plan current while executing.
6. Validate and summarize the result.
