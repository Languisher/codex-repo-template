# Release Workflow

1. Confirm the release scope and previous release point.
2. Review merged changes, migration notes, and known issues.
3. Update version metadata only according to project rules.
4. Prepare release notes grouped by user impact.
5. Run release validation.
6. Document rollback notes and post-release checks.

## Release Notes Skeleton

```markdown
## Added

## Changed

## Fixed

## Deprecated

## Removed

## Security

## Migration Notes

## Known Issues
```
